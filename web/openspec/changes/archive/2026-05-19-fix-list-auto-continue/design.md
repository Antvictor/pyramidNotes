## Context

`autoList.ts` 插件负责在列表项后按回车时自动继续生成下一个列表项标记。当前实现存在两个问题：
1. 正则表达式不匹配 `1)` 格式（有序列表的括号形式）
2. 没有实现回车两次取消列表的功能

## Goals / Non-Goals

**Goals:**
- 回车自动继续列表（支持 `-`、`+`、`1.`、`1)` 格式）
- 回车两次取消列表（第二次回车生成普通段落）

**Non-Goals:**
- 不修改列表项的嵌套行为
- 不处理任务列表（`- [ ]`）

## Decisions

### 1. 正则表达式扩展

**决定**：扩展正则以支持 `1)` 格式

```typescript
// 当前正则
/^(\s*[-*+]|\s*\d+\.)\s/

// 修改后
/^(\s*[+\-*]|\s*\d+\.?|\s*\d+\))\s/
```

### 2. 回车取消列表逻辑

**决定**：当列表项内容为空时，取消列表继续

```typescript
// 如果当前行是空列表项（只有标记没有内容）
// 第一次回车：清空当前行，开始普通段落
// 否则继续列表
```

## Risks / Trade-offs

- **Risk**: 空行检测可能与普通段落编辑冲突
- **Mitigation**: 只在列表项标记后无内容时触发取消
