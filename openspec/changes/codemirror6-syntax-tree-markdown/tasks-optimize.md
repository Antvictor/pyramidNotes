# Markdown 预览优化任务

## Issue 1: 无序列表不显示黑色 ·

**现象**：无序列表当前显示原始的 `-` 或 `+`，没有转化为黑色圆点 `•`。

**任务**：CSS 设置 `list-style-type: disc` for 无序列表内容

---

## Issue 2: 有序列表不显示序号

**现象**：有序列表应显示 1. 2. 3. 或 1) 2) 3)，当前显示原始输入。

**任务**：CSS 设置 `list-style-type: decimal` for 有序列表内容

---

## Issue 3: 列表回车不自动新增

**现象**：输入 `- item` 后按回车，不会自动新增下一项的标记。

**任务**：
- [ ] 检查 autoList.ts 插件是否正确加载
- [ ] 测试正则匹配 `- `、`+ `、`1.`、`1)` 格式
- [ ] 回车两次取消列表（检测空行或非列表行）

---

## Issue 4: 代码块被分割成多个单行块

**现象**：代码块内容每行都被当作独立的块处理，而不是整个代码块在同一个框内。

**逻辑**：` ``` ` 围栏明确标记开始和结束，围栏内所有内容是一个整体。

**根因**：FencedCode 节点可能包含整个代码块，但遍历时错误地处理了子节点。

**任务**：
- [ ] 检查 FencedCode 节点结构
- [ ] 确保 CSS `.md-code-block-content` 正确包裹内容

---

## Issue 5: 引用块被分割成多个单行块

**现象**：连续的 `> quote` 行没有合并到同一个引用框内。

**逻辑**：没有明确的结束标记，连续的 `>` 行隐式合并为一个引用块（与代码块不同）。

**根因**：Lezer 解析多行引用为嵌套结构（Blockquote > Paragraph），但遍历时没有正确识别。

**任务**：
- [ ] 检查 Lezer 对多行引用块的解析结构
- [ ] 确保 CSS `.md-blockquote-content` 正确合并连续引用行
