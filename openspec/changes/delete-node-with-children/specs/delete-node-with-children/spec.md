## ADDED Requirements

### Requirement: 删除空节点直接删除
当节点没有子节点时，删除操作直接执行，无需额外确认。

#### Scenario: 删除没有子节点的节点
- **WHEN** 用户删除一个没有子节点的节点
- **THEN** 节点直接从数据库和 UI 中删除
- **AND** 不弹出任何确认对话框

### Requirement: 删除有子节点的节点弹出选择对话框
当节点有子节点时，删除操作弹出对话框让用户选择处理方式。

#### Scenario: 删除有子节点的节点
- **WHEN** 用户尝试删除一个有子节点的节点
- **THEN** 弹出对话框，显示节点名称和子节点数量
- **AND** 提供三个选项：删除整个子树、仅删除父节点、取消

#### Scenario: 选择删除整个子树
- **WHEN** 用户在对话框中选择"删除整个子树"并点击确认
- **THEN** 删除选中节点及其所有子节点
- **AND** 从数据库中删除所有相关记录
- **AND** 删除所有相关的 markdown 文件

#### Scenario: 选择仅删除父节点
- **WHEN** 用户在对话框中选择"仅删除父节点"并点击确认
- **THEN** 将所有子节点的 `top` 字段更新为当前节点的 `top`（祖父节点）
- **AND** 删除选中节点
- **AND** 子节点在 UI 上正确显示在新的父节点下

#### Scenario: 选择取消
- **WHEN** 用户在对话框中点击取消
- **THEN** 不执行任何删除操作
- **AND** 关闭对话框

### Requirement: 子节点提升后层级正确
当子节点提升到祖父节点下时，其层级关系正确更新。

#### Scenario: 子节点提升到祖父节点下
- **WHEN** 节点 A（ID: aaa）被删除，其子节点 B（ID: bbb）和 C（ID: ccc）的 `top` 字段原为 `aaa`
- **AND** A 的 `top` 字段为 `xxx`（祖父节点）
- **WHEN** 用户选择"仅删除父节点"
- **THEN** B 和 C 的 `top` 字段更新为 `xxx`
- **AND** B 和 C 现在是 `xxx` 的子节点
