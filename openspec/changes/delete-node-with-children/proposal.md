## Why

删除有子节点的节点时，如果没有妥善处理子节点的归属，会导致子节点变成孤儿，在 UI 上显示异常或报错。用户需要明确的控制权来决定如何处理这种情况。

## What Changes

- 删除节点时，如果节点没有子节点，直接删除
- 删除节点时，如果节点有子节点，弹出对话框让用户选择处理方式：
  - **删除整个子树**：删除选中节点及其所有子节点（不可逆）
  - **仅删除父节点**：删除选中节点，子节点自动提升一级（挂到祖父节点下）
  - **取消**：不执行任何操作

## Capabilities

### New Capabilities
- `delete-node-with-children`: 增强删除节点逻辑，支持子节点处理策略

### Modified Capabilities
- 无

## Impact

- 涉及 `MindMap.jsx` 中 `deleteNode` 函数的修改
- 涉及确认对话框组件的使用（可能复用现有 `ConfirmDialog`）
- 涉及数据库更新逻辑：子节点提升时需要更新 `top` 字段
